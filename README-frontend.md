# Frontend assets (簡易ドキュメント)

このプロジェクトでフロントエンドの主要な変更点は以下です。

## 目的

- インラインで記述されていた CSS/JS を外部ファイルに統合して、保守性を向上。

## 変更したファイル

- `resources/views/index.blade.php`

  - インラインの `<style>` と `<script>` を削除。
  - 下記の共通アセットを読み込むように修正。

- `public/css/site.css` (新規)

  - 旧 `public/css/index.css` の内容を整理して移植。
  - サイト全体で使える共通スタイルを格納。

- `public/js/site.js` (新規)

  - 旧 `public/js/index.js` の内容（桜の Canvas エフェクト）を移植。
  - 追加の初期化処理や共通関数をここにまとめる想定。

## 使い方

- 開発中は `php artisan serve` などでローカルサーバを起動し、ブラウザで `/` を開くと反映されます。

- 画像等は `public/storage/...` に配置してください（既存コードが参照）。

## 備考 / 今後の改善点

- Vite / Mix を導入してアセットのビルド・バージョニングを行うと CD での配布が容易になります。

- 他のビューに同様のインライン資産があれば、同手順で外部化を推奨します。

- site.js 内の petalCount や描画パラメータは環境に合わせて調整可能です。

変更はワークスペース内のファイルに直接適用済みです。動作確認や追加の整備（Lint、ミニマムテストなど）を希望する場合は指示してください。
